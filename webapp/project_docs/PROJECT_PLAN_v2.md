# RetinaScope-AI — Project Plan v2.0

### A Diabetic Retinopathy Screening Web Application with Vessel Segmentation, DR Grading, and LLM-Driven Vascular Damage Analysis

> **v2 Change Summary (vs v1.0)**
> 1. Lesion segmentation model **removed from webapp** (retained in research pipeline only; webapp uses vessel + grader only).
> 2. Visualization upgraded from 4-panel to **5-panel dual-mask viewer**: clean binary mask + LLM-annotated damage mask (red ellipses on full mask + expandable zoom crops).
> 3. New **Vessel Damage Detection sub-system**: OpenCV skeletonization → connectivity/tortuosity analysis → Claude Vision annotates damaged regions → backend renders red ellipses → returns both panels.
> 4. LLM-2 (Vascular Damage Analyst) now **drives the visual annotation**, not just the text report — analysis and rendering are unified.
> 5. Minor design refresh notes added (§8).

---

## 1. Project Overview

**RetinaScope-AI** is a clinical-decision-support web application that ingests a single retinal fundus image and produces:

1. A **clean binary vessel segmentation mask** — pure black background, white vessels, nothing else (U-Net ensemble, Dice = 0.87).
2. A **damage-annotated vessel mask** — same mask with red ellipses drawn around detected damaged/discontinuous vessel regions, identified by LLM vision analysis.
3. A **DR grade** on the standard 0–4 ICDR scale (CNN grading classifier, QWK = 0.96).
4. A **calibrated confidence score** plus **closeness-to-next-grade** (e.g., "75 % confident — 17 % toward Grade 1").
5. A **structured clinical narrative** generated by a *Reasoning LLM* (LLM-1): medical explanation, risk factors, recommendations, lifestyle and follow-up advice.
6. A **vascular damage report** generated by a *Vision-Conditioned LLM* (LLM-2): localized vessel-damage findings grounded in quantitative biomarkers, with a "needs specialist review" flag.
7. A **consultation chat** — streaming Q&A scoped to the current case.

> **Why vessel-only for the webapp?**
> The lesion segmentation model (MA, HE, EX, OD, CW) adds significant inference latency and its current mean Dice (~0.56) is not yet at clinical presentation quality. The vessel model (Dice 0.87) and grader (QWK 0.96) already tell the complete DR screening story. Lesion masks will be re-integrated in v3 once the segmentation improvement plan reaches the target mean Dice ≥ 0.67.

## 5. Inference Pipeline (request lifecycle)

1. **Upload** — frontend POSTs `multipart/form-data` to `/api/v1/analyze`.

2. **Preprocessing**
   - Crop to circular field-of-view (Otsu threshold on green channel).
   - Apply **CLAHE** on the L-channel (LAB space).
   - Resize to model input size (512×512 — read from `preprocess.json`).
   - Normalize with the exact mean/std used during training.

3. **Parallel inference** (`asyncio.gather`)
   - **Vessel U-Net** → `prob_map ∈ [0,1]^{H×W}` → threshold @ 0.5 → `binary_mask`.
   - **Grader CNN** → logits → temperature-scaled softmax → `grade`, `probs[5]`.
   - *(Lesion U-Net removed from webapp inference path.)*

4. **Post-processing — Panel A (Clean Mask)**
   - Morphological cleanup of mask (remove < 25-px isolated components).
   - Render **Panel A**: pure black background (`0,0,0`), white vessel pixels (`255,255,255`) — no borders, no overlays, no labels. This is the clean scientific output.

5. **Vessel Damage Analysis — Panel B** *(new sub-system, see §5a)*
   - Run OpenCV skeletonization + connectivity analysis on the binary mask.
   - Compute per-segment tortuosity and detect discontinuities.
   - Send the clean mask (base64) + numeric biomarkers to **LLM-2 (Claude Vision)**.
   - LLM-2 returns JSON bounding boxes of damaged regions (see §6).
   - Backend draws **red ellipses** (not filled, stroke only) on a copy of the clean mask at the returned coordinates.
   - Render **Panel B**: damage-annotated mask.
   - Generate **zoom crop thumbnails**: top-3 highest-severity bounding boxes → 96×96 px crops, stored as a separate list for the `DamageZoomStrip` component.

6. **Visualization panels** (see §8 for full layout):
   - Panel 1: Original fundus (CLAHE preprocessed)
   - Panel 2: Probability heatmap (jet colormap, colorbar 0–1)
   - Panel 3: Panel A — clean binary vessel mask
   - Panel 4: Panel B — damage-annotated mask (red ellipses)
   - Panel 5: Red overlay (vessels on original fundus)

7. **Confidence layer**
   - Calibrated probability (temperature-scaled softmax).
   - Closeness-to-next-grade: `probs[grade+1] / (probs[grade] + probs[grade+1])`.
   - Epistemic uncertainty: 20-pass MC-Dropout → predictive entropy.
   - Decision rule: entropy < 0.3 and top-1 > 0.85 → **High confidence**; 0.3–0.6 → **Second look advised**; ≥ 0.6 or boundary → **Refer to specialist**.

8. **LLM Layer** (see §6).

9. **Persist** — case, images, masks, annotated mask, biomarkers, reports → DB + MinIO.

10. **Respond** — single JSON (see §7).

### 5a. Vessel Damage Detection Sub-System (NEW)

This is the core new engineering piece that powers Panel B. It runs entirely in Python before the LLM call, feeding the LLM quantitative evidence to anchor its visual analysis.

```python
# core/vessel_analysis.py

import cv2
import numpy as np
from skan import skeleton_to_csgraph, Skeleton
from skimage.morphology import skeletonize

def analyze_vessel_damage(binary_mask: np.ndarray) -> dict:
    """
    Returns a dict ready to be injected into the LLM-2 prompt as JSON.
    """
    # 1. Skeletonize the vessel mask (thin to 1-pixel centerlines)
    skeleton = skeletonize(binary_mask > 0)

    # 2. Build graph from skeleton — finds branch points and endpoints
    sk = Skeleton(skeleton)

    # 3. For each vessel segment, compute:
    #    - path length vs Euclidean distance → tortuosity index
    #    - endpoint density → high density = many broken segments = damage
    segment_data = []
    for i in range(sk.n_paths):
        coords = sk.path_coordinates(i)
        if len(coords) < 5:
            continue   # skip micro-fragments
        path_len = np.sum(np.linalg.norm(np.diff(coords, axis=0), axis=1))
        eucl_len = np.linalg.norm(coords[-1] - coords[0])
        tortuosity = path_len / max(eucl_len, 1.0)
        # bounding box of this segment
        y_min, x_min = coords.min(axis=0)
        y_max, x_max = coords.max(axis=0)
        segment_data.append({
            "tortuosity": round(float(tortuosity), 3),
            "length_px": round(float(path_len), 1),
            "bbox": [int(x_min), int(y_min), int(x_max), int(y_max)]
        })

    # 4. Connectivity — count isolated components (= broken vessel count)
    num_components, labels = cv2.connectedComponents(
        (binary_mask > 0).astype(np.uint8)
    )

    # 5. Quadrant vessel density (NW, NE, SW, SE)
    H, W = binary_mask.shape
    quadrant_density = {
        "NW": float(binary_mask[:H//2, :W//2].mean()),
        "NE": float(binary_mask[:H//2, W//2:].mean()),
        "SW": float(binary_mask[H//2:, :W//2].mean()),
        "SE": float(binary_mask[H//2:, W//2:].mean()),
    }

    # 6. Flag high-tortuosity segments (>1.5 is abnormal)
    tortuous = [s for s in segment_data if s["tortuosity"] > 1.5]

    return {
        "num_vessel_components": num_components - 1,   # -1 for background
        "num_broken_segments_estimate": max(0, num_components - 1 - 20),
        "high_tortuosity_segments": tortuous[:20],  # top 20 worst
        "quadrant_density": quadrant_density,
        "total_vessel_segments": len(segment_data),
    }


def draw_damage_annotations(
    clean_mask: np.ndarray,
    damage_bboxes: list[dict]   # from LLM-2 JSON response
) -> np.ndarray:
    """
    Draws red ellipses on a copy of the clean mask at each
    damage region returned by LLM-2.

    Each bbox dict: {"x_min":int, "y_min":int, "x_max":int, "y_max":int,
                     "severity":"low"|"medium"|"high", "finding": str}
    """
    annotated = cv2.cvtColor(clean_mask, cv2.COLOR_GRAY2BGR)
    color_map = {"low": (0,165,255), "medium": (0,100,255), "high": (0,0,255)}
    thickness_map = {"low": 1, "medium": 2, "high": 3}

    for bbox in damage_bboxes:
        x1, y1, x2, y2 = bbox["x_min"], bbox["y_min"], bbox["x_max"], bbox["y_max"]
        severity = bbox.get("severity", "medium")
        cx, cy = (x1 + x2) // 2, (y1 + y2) // 2
        rx, ry = max((x2 - x1) // 2, 10), max((y2 - y1) // 2, 10)
        color = color_map.get(severity, (0, 100, 255))
        thick = thickness_map.get(severity, 2)
        cv2.ellipse(annotated, (cx, cy), (rx, ry), 0, 0, 360, color, thick)
        # Small severity label above ellipse
        label = f"{severity[0].upper()} — {bbox.get('finding','')[:20]}"
        cv2.putText(annotated, label, (x1, max(y1-6, 10)),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.35, color, 1)

    return annotated
```

**Why red ellipses, not zoom-ins (reasoning):**
Zoom-ins lose the spatial reference that clinicians need — an ophthalmologist must know *which quadrant* and *which vessel tree* is affected, not just what the damage looks like up close. Red ellipses on the full mask preserve spatial anatomy while drawing the eye to pathology. Zoom crops of the top-3 severity regions are provided as *supplementary* thumbnails in the `DamageZoomStrip` component — the best of both approaches.

## 6. LLM Layer Design

### LLM-1 — Clinical Explainer (unchanged from v1)

- **Inputs**: grade, calibrated probabilities, closeness score, vascular biomarkers, patient metadata (age, T1/T2, HbA1c if provided).
- **Outputs (strict JSON)**:
  ```json
  { "summary": "...", "pathophysiology": "...", "risk_factors": [...],
    "recommendations": [...], "follow_up_window": "...",
    "lifestyle_advice": [...], "red_flags": [...], "disclaimer": "..." }
  ```
- Prompt grounded in ICDR + AAO Preferred Practice Pattern. Temperature 0.2.

### LLM-2 — Vascular Damage Analyst (UPDATED — now drives visual annotation)

This is the central change in v2. LLM-2 now serves a **dual role**: it produces the text vascular report AND the bounding box coordinates that the backend uses to draw Panel B.

**Inputs sent to Claude Vision:**
1. `image`: The **clean binary vessel mask** (Panel A) as base64 PNG — pure black/white, no decorations.
2. `biomarkers_json`: Numeric output of `analyze_vessel_damage()` — tortuosity segments, component counts, quadrant densities.
3. `grade`: DR grade from the grader CNN.

**Prompt strategy (`vascular_damage_analyst.txt`):**

```
You are a retinal vascular analyst assistant. You will be shown a binary vessel segmentation mask 
(white vessels on black background) from a diabetic retinopathy screening pipeline.

You are also given these quantitative vessel metrics computed from the mask:
{biomarkers_json}

The patient's CNN-predicted DR grade is: {grade}

Your task is to identify regions of the mask where the vessel structure appears:
- DISCONTINUOUS (vessel segments that are broken, have gaps, or abruptly end)
- ABNORMALLY TORTUOUS (vessels that curve or loop unnaturally)  
- CALIBER IRREGULAR (vessels that are unusually thick, thin, or have abrupt width changes)
- FOCALLY ABSENT (regions of the retina where vessels should be present but are missing)

Return ONLY a JSON object matching this exact schema — no preamble, no markdown:
{
  "damaged_regions": [
    {
      "x_min": int,   // pixel coordinates in 512x512 mask
      "y_min": int,
      "x_max": int,
      "y_max": int,
      "severity": "low" | "medium" | "high",
      "finding": string   // ≤ 40 chars, e.g. "Vessel discontinuity — superior arcade"
    }
  ],
  "overall_damage_score": int,  // 0–100
  "per_grade_severity": { "0": float, "1": float, "2": float, "3": float, "4": float },
  "needs_specialist_review": bool,
  "rationale": string,          // 1–2 sentences grounded in the biomarkers
  "closeness_to_next_grade": float  // 0.0–1.0
}

Return at most 8 damaged_regions. If no damage is visible, return an empty array.
```

**Backend flow after LLM-2 response:**
```
LLM-2 JSON response
      │
      ├─→ vascular_report (to frontend text panel)
      │
      └─→ damaged_regions[].bbox
                │
                ▼
         draw_damage_annotations(clean_mask, bboxes)
                │
                ▼
         Panel B (damage-annotated mask PNG)
                │
                ├─→ sent to frontend as base64
                │
                └─→ top-3 severity bboxes → zoom crops (96×96 px)
                        │
                        └─→ DamageZoomStrip thumbnails
```

**Cross-check rule:** If `overall_damage_score > 60` AND CNN grade ≤ 1 → auto-flag "Specialist review recommended — vascular analysis disagrees with grade."

## 10. Critical Engineering Notes

- **Match training-time preprocessing exactly.** Save `preprocess.json` next to every `.pth` (input size, mean, std, color space, CLAHE params).
- **Clean mask must be pure black/white.** No gray antialiasing, no borders, no labels. Use `threshold=0.5` on the raw prob_map and output as 8-bit grayscale PNG — the frontend converts to black-bg for display.
- **LLM-2 bounding boxes are pixel coordinates in the 512×512 mask space.** If the frontend displays masks at a different resolution, scale accordingly. Store raw 512×512 coordinates in the DB.
- **Don't trust raw softmax.** Fit temperature scaling on a validation split before exposing confidence to users.
- **Lock LLM outputs to JSON schemas** (`pydantic` + `instructor`). Free-text medical output is a liability.
- **Always render the disclaimer** on every panel, every PDF export, every API response.
- **GPU vs CPU fallback**: model registry auto-detects and warms up on startup.

## 11. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| LLM bounding boxes hallucinated (no real damage) | Overlay bboxes only if `overall_damage_score > 15`; require ≥ 2 biomarker signals to confirm |
| LLM-2 returns bboxes outside image bounds | Clamp all coordinates to [0, 512] before drawing |
| Domain shift (camera/clinic not in training set) | OOD detector on image embeddings; "low confidence — refer" flag |
| Latency (2 models + 2 LLM calls) | Parallelize models with `asyncio`; cache LLM outputs by image hash |
| Overconfident grader near class boundaries | Temperature scaling + closeness score + automatic referral when entropy ≥ 0.6 |
| Regulatory exposure | "Research / decision support" framing, explicit disclaimer, no patient-facing diagnosis |
