# RetinaScope-AI

### A Diabetic Retinopathy Screening Web Application with Vessel Segmentation, DR Grading, and Dual-LLM Clinical Reasoning

---

## 1. Project Overview

**RetinaScope-AI** is a clinical-decision-support web application that ingests a single retinal fundus image and produces:

1. A **binary vessel segmentation mask** (U-Net based, Dice = 0.87).
2. A **lesion segmentation mask** (microaneurysms, hemorrhages, exudates).
3. A **DR grade** on the standard 0вҖ“4 ICDR scale (CNN classifier, QWK = 0.96).
4. A **calibrated confidence score** plus a **closeness-to-next-grade score** (e.g., "Grade 1 вҖ” 98 % toward Grade 2").
5. A **structured clinical narrative** generated by a *Reasoning LLM* (medical explanation, risk stratification, lifestyle and follow-up advice).
6. A **vascular-damage report** generated by a *Vision-Conditioned LLM* (localized vessel-damage findings, per-grade severity score, and a "needs specialist review" flag).

The deliverable visualization mimics the reference figure you provided:
`Original Fundus | Binary Mask | Probability Heatmap | Red Overlay`,
extended with a clinical report panel and a chat-style consultation drawer.

## 5. Inference Pipeline (request lifecycle)

1. **Upload** вҖ” frontend POSTs `multipart/form-data` to `/api/v1/analyze`.
2. **Preprocessing**
   - Crop to circular field-of-view (Otsu threshold on green channel).
   - Apply **CLAHE** on the L-channel (LAB space).
   - Resize to model input (e.g., 512Г—512 for vessel U-Net, 384Г—384 for grader вҖ” match your training).
   - Normalize with the **exact mean/std used during training** (read from a sidecar JSON next to each `.pth`).
3. **Parallel inference** (asyncio gather)
   - Vessel U-Net вҶ’ `prob_map вҲҲ [0,1]^{HГ—W}` вҶ’ threshold @ 0.5 вҶ’ `binary_mask`.
   - Lesion U-Net вҶ’ multi-class lesion map.
   - Grader CNN вҶ’ logits вҶ’ temperature-scaled softmax вҶ’ `grade`, `probs[5]`.
4. **Post-processing**
   - Morphological cleanup of mask (remove < 25-px components).
   - Build the **4-panel composite** (`visualizer.py`) вҖ” pixel-for-pixel match of your reference figure.
   - Compute vascular biomarkers: vessel density, mean tortuosity, fractal dimension, AVR.
5. **Confidence layer**
   - **Calibrated probability**: temperature-scaled softmax.
   - **Closeness-to-next-grade**: `closeness = probs[grade] / (probs[grade] + probs[grade+1])` вҶ’ flip to "98 % toward Grade 2" semantics.
   - **Epistemic uncertainty**: 20-pass MC-Dropout вҶ’ predictive entropy.
   - **Decision rule**:
     - `entropy < 0.3` AND `top1 > 0.85` вҶ’ **High confidence**
     - `0.3 вүӨ entropy < 0.6` вҶ’ **Medium вҖ” second look advised**
     - `entropy вүҘ 0.6` OR borderline grade boundary вҶ’ **Refer to specialist**
6. **LLM Layer** (see В§6).
7. **Persist** вҖ” case, image, masks, biomarkers, reports вҶ’ DB + MinIO.
8. **Respond** вҖ” single JSON containing base64 PNGs + structured text.

## 6. LLM Layer Design

### LLM-1 вҖ” Clinical Explainer

- **Inputs**: grade, calibrated probabilities, closeness score, vascular biomarkers, lesion summary, patient metadata (age, T1/T2, HbA1c if provided).
- **Outputs (strict JSON schema)**:
  ```
  { summary, pathophysiology, risk_factors, recommendations,
    follow_up_window, lifestyle_advice, red_flags, disclaimer }
  ```
- **Prompt strategy**: system prompt grounded in **ICDR + AAO Preferred Practice Pattern**; few-shot exemplars per grade; temperature 0.2 for reproducibility.
- **Guardrails**: refuse to diagnose definitively; always include "non-diagnostic, decision support only" disclaimer; flag any contradiction between grade and biomarkers.

### LLM-2 вҖ” Vascular Damage Analyst

- **Inputs**: vessel binary mask, probability heatmap, lesion mask, computed biomarkers (density, tortuosity, AVR, fractal dim), grade.
- **Strategy**: convert the 4-panel figure to a base64 image and send to a **vision-capable LLM** (Claude 3.5 Sonnet or GPT-4o), accompanied by the numeric biomarkers вҖ” this anchors the model on quantitative facts and prevents purely visual hallucination.
- **Outputs (strict JSON)**:
  ```
  { damaged_regions: [{quadrant, severity, finding}],
    overall_damage_score: 0-100,
    per_grade_score: { "0":..,"1":..,"2":..,"3":..,"4":.. },
    closeness_to_next_grade: float,
    needs_specialist_review: bool,
    rationale: string }
  ```
- **Cross-check**: if LLM-2's `overall_damage_score` and the CNN grade disagree by > 1 grade band вҶ’ auto-escalate to "Refer to specialist".

### Consultation Chat (optional 3rd flow)

A streaming `/chat` endpoint scoped to the current case; system prompt re-injects the structured report so the LLM can never "forget" the patient context.

## 10. Critical Engineering Notes

- **Match training-time preprocessing exactly.** Save a `preprocess.json` next to every `.pth` (input size, mean, std, color space, CLAHE params). Mismatched normalization is the #1 cause of "great offline metrics, garbage in production".
- **Don't trust raw softmax.** A 0.96 QWK does not imply calibrated probabilities вҖ” fit temperature scaling on a validation split before exposing confidence numbers to users.
- **Lock LLM outputs to JSON schemas** (`pydantic` + `instructor` or function-calling). Free-text medical output is a liability.
- **Always render the disclaimer**: *"Decision support only вҖ” not a medical diagnosis."* Place it in every report, every PDF export, every API response.
- **PHI handling**: strip EXIF, hash filenames, encrypt at rest, audit every read.
- **GPU vs CPU fallback**: model registry should auto-detect and warm up on startup; first request must not pay the load cost.
- **Reproducibility**: pin `torch`, `torchvision`, `numpy`, `opencv-python` versions; commit `requirements.lock`.

## 11. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Domain shift (camera/ethnicity not in training set) | Out-of-distribution detector on image embeddings; "low-confidence вҖ” refer" flag |
| LLM hallucination of medical facts | Strict JSON schema + biomarker grounding + post-hoc fact filter |
| Overconfident grader near class boundaries | Temperature scaling + closeness score + automatic referral when entropy high |
| Latency (3 models + 2 LLM calls) | Parallelize models with `asyncio`; cache LLM outputs by image hash; optional ONNX/TensorRT export |
| Regulatory exposure | Frame as "research / decision support", explicit disclaimer, no patient-facing diagnosis |
